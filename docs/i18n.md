# Internationalization (i18n)

This starter kit includes **internationalization (i18n)** support using **next-intl**.
This allows you to build applications that support multiple languages easily.

---

# Why Use next-intl?

* Fully compatible with **Next.js App Router**
* Supports **Server and Client Components**
* Type-safe translations
* Easy to add new languages
* Works seamlessly with **React components**

---

# File Structure

Translation files are located in the `messages/` directory:

```id="j4krm1"
messages/
├ en.json
├ es.json
└ fr.json
```

Each JSON file represents a **language**.

Example `en.json`:

```json id="fbx1k3"
{
  "Index": {
    "title": "Welcome",
    "subtitle": "Manage your projects efficiently"
  },
  "Dashboard": {
    "greeting": "Hello, {name}!"
  }
}
```

# Adding a New Language

1. Create a new JSON file for the language:

```id="k2ldq0"
messages/de.json
```

2. Add translations:

```json id="z7r8q1"
{
  "Index": {
    "title": "Willkommen",
    "subtitle": "Verwalten Sie Ihre Projekte effizient"
  }
}
```

3. Update the list of supported locales in your configuration if needed:

```ts id="h5x1q3"
export const LOCALES = ["en", "es", "fr", "de"]
```

# Using Translations in Server Components

Use `getTranslations` from `next-intl/server`.

Example:

```ts id="w9v2k4"
import { getTranslations } from "next-intl/server"

export default async function Page() {
  const t = await getTranslations("Index")

  return <h1>{t("title")}</h1>
}
```

---

# Using Translations in Client Components

Use `useTranslations` from `next-intl`.

Example:

```tsx id="f8k1v7"
"use client"

import { useTranslations } from "next-intl"

export function DashboardGreeting({ name }: { name: string }) {
  const t = useTranslations("Dashboard")

  return <p>{t("greeting", { name })}</p>
}
```

---

# Dynamic Language Switching

Languages can be switched by:

1. Updating the locale in the URL (e.g., `/en`, `/es`)
2. Passing the locale to `getTranslations` or `useTranslations`

Example:

```ts id="y2p0f8"
const t = await getTranslations("Index", { locale: "es" })
```

---

# Middleware Support

You can use middleware to detect and redirect users based on their preferred language.

Example:

```ts id="m5h3v1"
import { NextRequest, NextResponse } from "next/server"

export function middleware(req: NextRequest) {
  const locale = req.nextUrl.pathname.split("/")[1] || "en"
  return NextResponse.next()
}
```

# Best Practices

### Group translations by page or feature

Keep translations organized by namespace, e.g., `Index`, `Dashboard`.

---

### Use placeholders for dynamic content

Example:

```json
{
  "Dashboard": {
    "greeting": "Hello, {name}!"
  }
}
```

---

### Keep JSON files consistent

Ensure all languages have the same keys to avoid runtime errors.

---

### Use server components for heavy translations

Server components allow preloading translations for better performance.
