# Database (Prisma)

This starter kit uses **Prisma** as the ORM to interact with a **PostgreSQL** database.

Prisma provides type-safe database access and works seamlessly with TypeScript and tRPC.

---

# Why Prisma?

* Type-safe queries
* Automatic TypeScript types
* Easy migrations
* Works well with Next.js
* Simple integration with tRPC and background jobs

---

# File Structure

Prisma files are located in the `prisma/` folder:

```id="d4kqj1"
prisma/
├ schema.prisma      # Database schema and models
├ migrations/        # Applied migrations
└ seed.ts            # Optional seed script
```

Backend usage lives in:

```id="x8o7f2"
src/server/db.ts
```

This file exports a Prisma client instance:

```ts id="t4m2q3"
import { PrismaClient } from "@prisma/client"

export const db = new PrismaClient()
```

---

# Defining Models

Models are defined in `schema.prisma`.

Example `User` model:

```prisma id="r9k3w5"
model User {
  id        String   @id @default(cuid())
  name      String?
  email     String?  @unique
  role      String?
  createdAt DateTime @default(now())
}
```

Example `Project` model:

```prisma id="k7h2v1"
model Project {
  id          String   @id @default(cuid())
  name        String
  description String?
  ownerId     String
  owner       User     @relation(fields: [ownerId], references: [id])
  createdAt   DateTime @default(now())
}
```

# Generating the Prisma Client

After modifying the schema, generate the Prisma client:

```bash id="j3m1t9"
npm run db:generate
```

This updates the TypeScript types for type-safe queries.

---

# Pushing Changes to the Database

For quick prototyping, push the schema directly:

```bash id="v5n2y7"
npm run db:push
```

This creates or updates tables without generating a SQL migration.

---

# Running Migrations

For production or structured changes, create migrations:

```bash id="w9p4l8"
npm run db:migrate
```

Migrations are stored in `prisma/migrations/` and can be applied in any environment.

---

# Seeding the Database

You can seed initial data using `prisma/seed.ts`:

```ts id="g4r2k5"
import { db } from "@/server/db"

async function seed() {
  await db.user.create({
    data: {
      name: "Admin",
      email: "admin@example.com",
      role: "ADMIN",
    },
  })
}

seed()
  .catch(console.error)
  .finally(() => process.exit())
```

Run the seed script:

```bash id="m9p1f7"
npm run db:seed
```

# Using Prisma in tRPC

The Prisma client is available in the **tRPC context**:

```ts id="h8v2k3"
ctx.db.project.findMany()
```

Example query:

```ts id="b7m9d2"
import { createTRPCRouter, publicProcedure } from "@/server/api/trpc"

export const projectRouter = createTRPCRouter({
  getAll: publicProcedure.query(({ ctx }) => {
    return ctx.db.project.findMany()
  }),
})
```

# Relationships

Prisma supports **one-to-one**, **one-to-many**, and **many-to-many** relationships.

Example one-to-many (User → Projects):

```prisma id="p3k2l1"
model User {
  id       String    @id @default(cuid())
  name     String?
  projects Project[]
}

model Project {
  id      String @id @default(cuid())
  name    String
  ownerId String
  owner   User   @relation(fields: [ownerId], references: [id])
}
```

---

# Query Examples

Fetch all projects for a user:

```ts id="z8v7f2"
const projects = await db.project.findMany({
  where: { ownerId: userId },
})
```

Create a new project:

```ts id="c9m8j1"
await db.project.create({
  data: {
    name: "New Project",
    ownerId: userId,
  },
})
```

Update a project:

```ts id="x7p3v4"
await db.project.update({
  where: { id: projectId },
  data: { name: "Updated Project Name" },
})
```

# Best Practices

### Use Prisma client from `src/server/db.ts`

Avoid creating multiple PrismaClient instances.

---

### Use migrations for production

`db:push` is only recommended for prototyping.

---

### Keep models normalized

Design relationships properly for scalability.

---

### Use type-safe queries

Leverage TypeScript types generated by Prisma.
